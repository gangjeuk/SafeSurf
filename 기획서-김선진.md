# Research Chrome Extension 프로젝트 기획서

## 1. 프로젝트 개요

### 1.1 프로젝트명
**Hypatia - AI 기반 리서치 지원 Chrome Extension**

### 1.2 프로젝트 목적
비즈니스 리서치 업무를 수행하는 실무자들이 체계적이고 효율적으로 조사를 수행할 수 있도록 지원하는 Chrome Extension 개발

### 1.3 프로젝트 배경
- 기존 리서치 작업은 1-2주의 시간이 소요되는 고강도 업무
- AI 기술의 발전으로 리서치 업무의 완성도가 높아지고 작업 시간이 단축됨
- 체계적인 리서치 방법론(인접/세부/확장/다각 조사)을 실무에 적용할 도구 필요
- AI 할루시네이션 문제를 교차 검증으로 해결하는 시스템 필요

## 2. 프로젝트 필요성

### 2.1 핵심 문제점
1. **비정형화된 리서치 프로세스**
   - "무엇을 찾아야 하는지" 명확하지 않은 경우가 많음
   - 조사 범위와 항목이 모호하여 불완전한 결과 도출

2. **시간 효율성 문제**
   - 전통적인 리서치 방식은 1-2주 소요
   - 반복적인 검색과 자료 수집에 과도한 시간 투입

3. **신뢰도 검증의 어려움**
   - 서로 다른 출처에서 상이한 데이터 발견 시 판단 기준 부재
   - 교차 검증 프로세스의 체계화 필요

4. **AI 활용의 한계**
   - AI 할루시네이션 문제로 인한 신뢰도 저하
   - 프롬프트 작성 방법에 따라 결과 품질 편차 큼

### 2.2 솔루션 방향
**체계적인 리서치 방법론 + AI 기술 + 검증 시스템**을 결합한 Chrome Extension 개발

## 3. 리서치 방법론 체계

### 3.1 4가지 핵심 조사 방식

#### 3.1.1 인접 조사 (Adjacent Research)
**정의**: 조사하려는 항목과 인접한 항목들을 함께 조사

**예시**:
- 매출 조사 시 → 매출 규모, 성장률, 영업이익, 영업이익률, 영업이익 성장률
- 업계별 지표: 거래액, 송금액, 결제액 등
- 매출 조사 시 '비용'도 인접 항목으로 고려

**적용 가치**: 맥락을 이해하고 연관 지표를 놓치지 않음

#### 3.1.2 세부 조사 (Detailed Research)
**정의**: 조사 대상을 더 작은 요소로 분해하여 조사

**예시**:
- 매출 → 사업부별, 서비스별, 부문별, 상품별, 지역별 매출
- 플랫폼 트래픽 → 탭별 트래픽, 요일별 트래픽

**적용 가치**: 구체적인 인사이트 도출 및 문제점 파악

#### 3.1.3 확장 조사 (Expanded Research)
**정의**: 조사 대상을 더 넓은 범위로 확장하여 비교

**비교 기준 설정**:
- **비교 대상**: 자사, 경쟁사, 글로벌 평균, 지역 평균, 국내 평균
- **시간 단위**: 분기, 반기, 연도
- **비교 항목**: 절대값, 상승률, 비율
- **시간 길이**: 현재, 3년, 5년

**예시**:
- 국내 시장 규모 → 타 국가 시장 규모, 지역별 시장 규모, 글로벌 시장 규모

**적용 가치**: 객관적인 위치 파악 및 벤치마킹

#### 3.1.4 다각 조사 (Multi-angle Research)
**정의**: 조사 대상은 고정하고 조사 항목을 다양화

**예시**:
- 플랫폼 조사 → DAU, MAU, PV, 세션, 세션 지속 시간

**세부 조사 vs 다각 조사**:
- 세부 조사: 대상을 분해 (예: 플랫폼 → 탭별로 분해)
- 다각 조사: 항목을 분해 (예: 트래픽 → DAU, MAU, PV로 분해)

### 3.2 방법론의 실무 적용
**리서치를 잘한다는 것** = 무엇을, 어디서 찾아, 어떻게 추리하는가

이 4가지 조사 방식의 조합을 통해 **무엇을 찾아야 하는지 정형화** 가능

## 4. 데이터 신뢰도 검증 체계

### 4.1 출처 신뢰도 판단 기준

#### 우선순위 1: 직접 연관 기관 우선
- A사 증권사 리포트 < A사 직접 발행 자료

#### 우선순위 2: 교차 검증 (Triangulation)
- 신뢰도 높은 기관 3곳 중 2곳이 근사한 수치 → 해당 수치 채택

#### 우선순위 3: 데이터 일관성 확보
- 특정 기관이 모든 조사 항목을 보유한 경우
- 해당 기관의 자료만 활용하여 일관성 유지
- 예: Statista vs eMarketer → 더 포괄적인 기관 선택

### 4.2 신뢰도 높은 출처 예시
- Gartner (테크 분야)
- Statista
- eMarketer
- 각 기업의 공식 IR 자료
- 정부/공공기관 통계

## 5. AI 활용 전략

### 5.1 AI 리서치의 장점
- **시간 효율**: 1-2주 작업을 수십 분으로 단축
- **완성도**: AI 리서치 + 검증이 인간 리서치 리뷰보다 빠름
- **현재 AI 작업 중 가장 높은 완성도**: 즉시 도입 가능

### 5.2 효과적인 프롬프트 작성법

#### 나쁜 예시
```
"휴대폰 시장을 조사해줘"
```

#### 좋은 예시
```
2020년부터 2025년까지의 글로벌 스마트폰 디바이스 시장 규모를 조사해 줘.

시장 규모에 대한 정의는 매년 실제로 판매된 스마트폰의 디바이스 수,
그리고 매출을 기반으로 조사해 주면 돼.

신뢰할 수 있는 출처를 활용해서 조사하고 조사한 출처에 대한 교차 검증을 통해
조사한 내용이 정확하도록 여러 차례 검증을 해 주어야 해.

추가적으로 제조사별 매출은 어느 정도였는지 파악해 주고
판매 대수 기준으로도 주요 플레이어들의 점유율을 연도별로 알려줘.

또한, 최종적으로 내가 검수할 수 있도록 정보를 찾은 출처를
반드시 세부적으로 표기해서 전달해 줘.
```

### 5.3 프롬프트 필수 3요소

1. **정확한 조사 범위와 항목**
   - 무엇을 조사할 것인가
   - 어떤 기간 동안 조사할 것인가
   - 조사 대상이 매출인지, 판매 수량인지 등 명확히 정의

2. **교차 검증 요청**
   - AI 스스로 데이터를 교차 검증하도록 명시
   - 사람의 검수 시간 단축

3. **검수용 출처 요청**
   - 할루시네이션 방지를 위한 안전장치
   - 출처를 반드시 세부적으로 표기하도록 요청

### 5.4 Deep Research 기능 활용
심층 리서치 기능을 적극 활용하여 더 정확하고 깊이 있는 조사 수행

## 6. Chrome Extension 기능 요구사항

### 6.1 핵심 기능

#### 6.1.1 리서치 템플릿 제공
- 4가지 조사 방식(인접/세부/확장/다각)을 선택하여 자동으로 프롬프트 생성
- 비교 기준 설정 UI (대상, 시간 단위, 항목, 기간)

#### 6.1.2 스마트 프롬프트 생성기
- 사용자 입력을 기반으로 3가지 필수 요소를 포함한 프롬프트 자동 생성
  - 정확한 조사 범위와 항목
  - 교차 검증 요청
  - 검수용 출처 요청

#### 6.1.3 출처 검증 시스템
- 수집된 데이터의 출처 자동 분류 및 신뢰도 평가
- 교차 검증 상태 시각화
- 상충되는 데이터 발견 시 알림

#### 6.1.4 리서치 결과 관리
- 조사 내용 저장 및 버전 관리
- 출처별 데이터 정리
- 보고서 형태로 내보내기

### 6.2 부가 기능

#### 6.2.1 신뢰 출처 데이터베이스
- Gartner, Statista 등 신뢰도 높은 기관 목록 관리
- 사용자 커스텀 출처 추가 가능

#### 6.2.2 AI 연동
- ChatGPT, Claude 등 주요 AI 서비스와 연동
- Deep Research 기능 직접 호출

#### 6.2.3 협업 기능
- 팀원 간 리서치 결과 공유
- 피드백 및 코멘트 기능

## 7. 기대 효과

### 7.1 정량적 효과
- **시간 단축**: 1-2주 → 수십 분 (약 90% 이상 단축)
- **검증 시간 단축**: 체계적인 교차 검증으로 검수 시간 50% 이상 단축
- **데이터 정확도 향상**: 교차 검증 시스템으로 할루시네이션 최소화

### 7.2 정성적 효과
- **리서치 품질 표준화**: 방법론 기반으로 일관된 품질 확보
- **의사결정 속도 향상**: 빠른 데이터 수집으로 신속한 판단 가능
- **업무 만족도 증가**: 반복적인 수작업 감소, 분석에 집중 가능

## 8. 개발 우선순위

### Phase 1: MVP (최소 기능 제품)
1. 4가지 조사 방식 템플릿 제공
2. 스마트 프롬프트 생성기

1. 출처 검증 시스템
2. 신뢰 출처 데이터베이스
3. 교차 검증 시각화

### Phase 3: 고도화
1. 리서치 결과 관리 시스템
2. 협업 기능
3. 보고서 자동 생성
